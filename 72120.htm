Web Server (Apache)

Les paramètres du serveur Web (sous Packs de services > sélectionnez un pack > onglet Serveur Web) vous permettent de définir la configuration du serveur Web pour tous les domaines qui seront créés sous un pack de services spécifique.

Remarque : les paramètres du serveur Web peuvent également être modifiés individuellement pour chaque domaine. Les modifications au niveau du pack de services n'écrasent pas les paramètres personnalisés des domaines existants. Par conséquent, les modifications que vous effectuez sous Packs de services > sélectionnez un pack > Serveur Web, sont effectives uniquement pour les domaines nouvellement créés.

Pour en savoir plus sur la configuration du serveur Web, consultez la section Serveur Web Apache (Linux).

 

Paramètres communs Apache.

Nous vous recommandons de sélectionner l'option Restreindre le suivi des liens symboliques afin d'augmenter le niveau de sécurité du serveur. Si cette option est sélectionnée, les utilisateurs de Plesk ne peuvent plus suivre les liens symboliques qui risquent de pointer vers des répertoires auxquels les utilisateurs ne doivent pas pouvoir accéder. Les utilisateurs ne peuvent ni utiliser / ni activer la directive FollowSymLink dans .htaccess.

Remarque : cette option risque d'interrompre de nombreuses applications (applications APS, mais aussi si un utilisateur à copié/collé un fichier .htaccess qui contient FollowSymLinks).

  

Directives pour HTTP et Directives pour HTTPS

Pour prédéfinir les directives Apache qui seront utilisées lors de l'accès au site Web via HTTP et HTTPS, utilisez les champs Directives pour HTTP et Directives pour HTTPS. Lorsque vous éditez les champs, utilisez la syntaxe comme dans httpd.conf. Par exemple, si vous souhaitez définir une page d'erreur personnalisée, ajoutez la ligne : 

ErrorDocument 401 /my_error_page.html

 

Paramètres Apache communs

Par défaut, Apache est utilisé avec N qui envoie par proxy des requêtes HTTP à Apache. Vous pouvez exclure Apache du processus de gestion des requêtes en contenu Web des sites Web et utiliser Nginx à cette fin. Pour ce faire, décochez l'option Mode proxy. Cette option sera désactivée sur les domaines nouvellement créés dans des abonnements synchronisés avec l'abonnement. Cela ne changera rien pour les domaines existants.

Pour plus d'informations, consultez la section Ajuster les paramètres du serveur Web Apache.

  

Directives Nginx

Pour prédéfinir les directives Nginx, utilisez le champ Directives Nginx. Lorsque vous éditez ce champ, utilisez la même syntaxe que pour nginx.conf. Par exemple, si vous souhaitez compresser toutes les requêtes par proxy avec gzip, ajoutez la ligne : 

gzip_proxied any;
