Utilitaire de réparation de Plesk : Système de fichiers

L'utilitaire de réparation de Plesk avec l'aspect  fs  vérifie l'intégrité et la structure du système de fichiers. Il est possible de vérifier les fichiers système (ceux qui sont essentiels pour Plesk) et/ou les fichiers des hôtes virtuels sur le serveur. Vous pouvez aussi vérifier les fichiers de domaines spécifiques.

Remarque : cet aspect s'applique uniquement à Plesk pour Linux.

L'utilitaire de réparation avec l'aspect  fs  permet de détecter et de résoudre les problèmes suivants :

  • Propriétaires de fichiers ou répertoires inattendus.
  • Droits de fichiers ou répertoires inattendus.
  • Attributs étendus de fichiers ou répertoires inattendus.

Il est également possible de détecter l'absence de fichiers ou répertoires obligatoires (sans résolution du problème).

Les options suivantes sont spécifiques à cet aspect :

  • -system  : vérifie uniquement les fichiers système. Les vérifications concernent un certain nombre de fichiers système essentiels au bon fonctionnement de Plesk.
  • -vhosts  : vérifie uniquement les fichiers des hôtes virtuels.

Si aucune de ces options n'est utilisée, les fichiers système et ceux des hôtes virtuels sont vérifiés.

Si un domaine précis est indiqué, seuls les fichiers de ce domaine seront vérifiés. Si toute autre option (-system, -vhosts) est utilisée dans ce cas, elle sera ignorée.

Exemples :

  • plesk repair fs  : vérifie à la fois les fichiers des hôtes virtuels et les fichiers système sur le serveur.
  • plesk repair fs -vhosts  : vérifie uniquement les fichiers des hôtes virtuels sur le serveur.
  • plesk repair fs –system : vérifie uniquement les fichiers système sur le serveur.
  • plesk repair fs example.com : vérifie les fichiers de l'hôte virtuel  exemple.com  uniquement.
  • plesk repair fs example.com -vhosts : vérifie uniquement les fichiers de l'hôte virtuel  exemple.com . Dans ce cas, l'option –vhosts est ignorée.
  • plesk repair fs -system -v -n : donne des informations détaillées sur tout problème détecté dans le système de fichiers sans les résoudre.
