Configurer les paramètres de sauvegarde de serveurs SQL distants

Si vous voulez sauvegarder les bases de données hébergées sur un serveur Microsoft SQL Server à distance que vous avez enregistré dans Plesk auparavant, vous devez configurer les paramètres de sauvegarde pour ce serveur Microsoft SQL Server.

Pour configurer les paramètres de sauvegarde pour un serveur de bases de données distant Microsoft SQL Server, procédez comme suit :

  1. Allez dans l'onglet Outils & Paramètres > Serveurs de bases de données, puis cliquez sur le nom de serveur Microsoft SQL distant.
  2. Indiquez le répertoire physique temporaire pour le serveur SQL distant dans le champ Répertoire temporaire. Par exemple : C:\db_pleskbackup.

    Ce répertoire physique temporaire est requis pour la sauvegarde et la restauration des bases de données de serveur SQL distantes : Microsoft SQL Server place temporairement les fichiers de sauvegarde dans ce répertoire pendant la sauvegarde et Plesk place temporairement les fichiers de sauvegarde dans ce répertoire pendant la restauration. Vous devez créer ce répertoire sur le serveur distant et le rendre accessible pour la lecture et l'écriture par le serveur MS SQL.

  3. Partagez le répertoire physique créé à l'étape précédente et précisez le chemin d'accès du réseau vers ce répertoire dans le champ Répertoire réseau temporaire. Par exemple : \\ServerName\db_pleskbackup.

    Le serveur Plesk utilisera ce répertoire pour télécharger et envoyer les fichiers de sauvegarde temporaires. Vous devez créer le répertoire réseau sur le serveur distant. Montez-le sur un répertoire physique temporaire créé à l'étape précédente. Ensuite, rendez-le accessible via le réseau à l'utilisateur que vous créerez à l'étape suivante.

  4. Créez un utilisateur sur le serveur distant (par exemple : ServerName\db_backup_user). Donnez à cet utilisateur l'accès au répertoire réseau temporaire avec les droits de lecture / d'écriture.

    Le serveur Plesk utilisera cet utilisateur pour accéder au répertoire temporaire.

  5. Indiquez le nom de l'utilisateur et le mot de passe requis pour accéder au répertoire réseau temporaire.
  6. Cliquez sur OK.
