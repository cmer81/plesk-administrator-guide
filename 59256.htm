Sauvegarder et restaurer

Avec les fonctions de restauration et de sauvegarde des données fournies par Plesk, vous pouvez :

  • Sauvegarder tout le serveur.
  • Sauvegarder des comptes utilisateur individuels avec des sites Web.
  • Sauvegarder des abonnements individuels.
  • Planifier des sauvegardes.
  • Restaurer les données à partir des archives de sauvegarde.

Dans le tableau ci-dessous, vous trouverez un aperçu des objets inclus dans les sauvegardes à différents niveaux.

  • Paramètres et configuration des services système, comptes, sites, bases de données, boîtes mail, listes de diffusion.
  • Certificats SSL/TLS.

Stockage des sauvegardes

Les sauvegardes peuvent être stockées à deux emplacements :

  • Le stockage interne de Plesk : le stockage d'une sauvegarde est situé sur votre serveur Plesk.
  • Le répertoire FTP externe : le stockage d'une sauvegarde est situé sur un serveur externe sur le Web ou dans votre réseau local. Dans ce cas, vous devez fournir à Plesk le nom d'hôte du serveur et les identifiants d'accès utilisateur FTP.
Sauvegardes complètes et incrémentales

Plesk prend en charge deux types de sauvegardes :

  • Sauvegarde complète. Une sauvegarde complète inclut toutes les données, à chaque sauvegarde, quelle que soit la date de la dernière mise à jour des données.
  • Sauvegarde incrémentale. Une sauvegarde incrémentale contient uniquement les données modifiées depuis la date de création de la dernière sauvegarde.

Ce type de sauvegarde permet de réduire à la fois la durée de la sauvegarde et l'espace disque occupé par les fichiers de sauvegarde.

Pour plus d'informations, consultez la section sur la sauvegarde incrémentale.

Sauvegardes créées par vos clients

Parmi vos clients, ceux qui ont le droit d'utiliser les fonctionnalités de sauvegarde et de restauration peuvent procéder eux-mêmes à la sauvegarde et à la restauration des paramètres de leurs propres comptes et sites Web via le Panneau Client. Vos clients, vos revendeurs et les clients de ces derniers trouveront des raccourcis vers leurs répertoires dans leur Panneau Client (Sites Web & Domaines > Gestionnaire de sauvegardes).

Les utilisateurs de Plesk peuvent voir le rôle d'un utilisateur qui a créé une sauvegarde (administrateur, client ou revendeur) dans la liste des tâches de sauvegarde (Outils & Paramètres > Gestionnaire de sauvegardes). Cela permet aux clients de faire la différence entre les sauvegardes qu'ils ont créées eux-mêmes et les sauvegardes techniques de leur abonnement. Les sauvegardes techniques ont lieu lorsque les administrateurs ou revendeurs sauvegardent les abonnements des clients en les intégrant dans une sauvegarde plus importante. Par exemple, si l'administrateur de Plesk crée une sauvegarde niveau serveur, tous les abonnements des clients sont également sauvegardés et apparaissent pour les clients comme sauvegardes d'abonnements créées par l'administrateur.

En cas d'échec d'une tâche de sauvegarde, Plesk affiche une description détaillée de l'erreur dans un champ distinct d'une tâche de sauvegarde.

Création de sauvegardes protégées par un mot de passe

Depuis Plesk 11.0, vous pouvez protéger les données sensibles dans vos sauvegardes par un mot de passe. Dans ces conditions, aucun attaquant ne peut accéder aux données de votre sauvegarde si la sécurité de votre stockage de sauvegarde venait à être compromise.


Vous pouvez définir un mot de passe pour la sauvegarde dans les cas suivants :

  • Dans les paramètres de votre stockage FTP personnel (Sites Web & Domaines > Gestionnaire de sauvegardes > Paramètres du stockage FTP).
  • Lors du téléchargement d'un fichier de sauvegarde à partir du répertoire interne de Plesk vers un emplacement externe.

Lorsque vous réessaierez d'envoyer ces sauvegardes vers Plesk et de les restaurer, vous serez invité à indiquer le mot de passe utilisé pour la protection.
