Suppression de la fonction Mail du Panneau Client

Pour utiliser un serveur de messagerie fonctionnant sur une machine distincte ou pour interdire à vos utilisateurs d'exécuter des services de messagerie, vous pouvez supprimer de Plesk toutes les commandes liées à la gestion des services de messagerie et à l'ajout de nouveaux comptes mail. Pour cela, désactivez l'option Activer les fonctions de gestion de messagerie dans Plesk.

Cette option ne désactive pas vraiment le serveur de messagerie géré par Plesk ; elle supprime uniquement certains éléments de l'interface du Panneau Client. Ces éléments seront masqués pour les clients et les utilisateurs du service d'hébergement. Les éléments suivants sont supprimés :

  • Section Mail.
  • Section Utilisateurs > nom d'utilisateur> Changer les paramètres > option Créer une adresse mail sous votre compte.

Pour masquer les éléments de l'interface utilisateur associés aux services de messagerie depuis le Panneau Client :

  1. Dans le Panneau d'administration du serveur, allez dans Outils & Paramètres > Paramètres du serveur de messagerie (groupe Mail).
  2. Désactivez l'option Activer les fonctions de gestion de messagerie dans Plesk et cliquez sur OK.

      

Vous pouvez également masquer les fonctionnalités liées à la messagerie ainsi que les droits correspondants à l'aide du fichier /usr/local/psa/admin/conf/panel.ini. Pour ce faire, ajoutez la ligne suivante dans le fichier :

services.withoutMailService = true
