Utiliser des serveurs DNS externes

Bien que Plesk fournisse tous les instruments pour exécuter le DNS sur votre serveur, vous pouvez également héberger les zones DNS sur un serveur DNS externe. Cela peut être sur votre propre serveur ou sur un service DNS tiers comme Amazon Route 53 (http://aws.amazon.com/fr/route53/) ou DynECT (http://dyn.com/dns/dynect-managed-dns/).

Il y a deux situations possibles :

  • Vous n'utilisez qu'un serveur DNS externe.

    Vous pouvez exclure le composant Serveur DNS de votre installation Plesk. Pour cela, utilisez une installation de type personnalisé (voir le Guide de déploiement) et désélectionnez le composant correspondant (Prise en charge du serveur DNS BIND sous Linux et Serveur DNS Microsoft sous Windows). Dans ce cas, vous ne pouvez pas gérer les zones DNS via Plesk. Au lieu de cela, utilisez un serveur DNS.

  • Vous pouvez utiliser à la fois un serveur DNS pour stocker les informations sur les zones et le serveur DNS pour gérer les zones via Plesk.

    Si vous utilisez des services DNS externes, alors vous n'avez pas besoin de désactiver le serveur DNS local. Le serveur DNS local peut être activé ou désactivé : dans les deux cas, la gestion des zones s'effectue sur le serveur DNS de Plesk et les modifications dans les zones sont propagées au DNS externe.

    Par défaut, Plesk ne peut pas propager automatiquement les modifications des zones DNS aux serveurs DNS externes. Pour que cela fonctionne, vous devez écrire un script d'intégration. Ce script doit communiquer avec le backend du serveur DNS (comme l'API) et appliquer toutes les modifications des zones DNS réalisées dans Plesk. Pour savoir comment préparer ce type de script, consultez le document Développer des extensions pour Plesk Onyx, section Intégration avec des services DNS tiers.

Lorsque vous utilisez un serveur DNS externe et qu'un serveur DNS local de Plesk est installé en même temps, toutes les fonctions de Plesk liées au DNS sont prises en charge et fonctionnent normalement, notamment :

  • La gestion des templates, zones et enregistrement DNS avec l'interface de Plesk, les utilitaires de ligne de commande et les requêtes API.
  • Les applications APS qui utilisent l'aspect DNS.
  • D'autres services qui utilisent le DNS, comme la protection anti-spam DKIM, SPF et DMARC. Pour en savoir plus, consultez la section Outils anti-spam.

Par conséquent, les utilisateurs peuvent toujours modifier les paramètres DNS des domaines, même si le service DNS local est désactivé sans que le serveur DNS local ne soit désinstallé. Ceci est vrai pour les domaines créés avant que vous ne désactiviez le service DNS local.

Intégration avec Amazon Route 53

Si vous voulez utiliser le service DNS Amazon Route 53 avec Plesk, vous pouvez installer l'extension correspondante. Depuis Plesk 12.0, cette extension est disponible dans le Catalogue des extensions de l'interface de Plesk.

Pour intégrer Plesk à Amazon Route 53 :

  1. Allez sur la page Accueil, dans Extensions > Catalogue des extensions.
  2. Repérez l'extension Amazon Route 53 et cliquez sur Installer.
  3. Dans la liste des extensions installées, cliquez sur le nom de l'extension (Amazon Route 53).
  4. Saisissez vos identifiants de sécurité Amazon et activez le service.

Remarque : quand vous utilisez Amazon Route 53, les zones DNS sont gérées par Plesk, sauf si vous désinstallez le composant Serveur DNS de Plesk.

Important : si vous ajoutez ou modifiez manuellement des zones sur un serveur DNS externe intégré à Plesk, comme Amazon Route 53, ces modifications ne sont pas visibles dans Plesk et peuvent être écrasées ou supprimées par Plesk. Vous devez gérer les zones uniquement sur le serveur DNS de Plesk. Le serveur DNS de Plesk agit en tant que serveur DNS primaire (maître) qui propage les modifications des zones DNS au serveur externe.

Si vous voulez modifier les zones DNS manuellement dans Amazon Route 53, vous devez supprimer le composant du serveur DNS de Plesk et son intégration Route 53 de Plesk.

Pour utiliser exclusivement Amazon Route 53 :

  1. Supprimez l'extension Amazon Route 53 (Extensions > sélectionnez Amazon Route 53 et cliquez sur Supprimer).
  2. Désinstallez les serveurs DNS de Plesk à l'aide du Programme d'installation de Plesk (Ajouter et supprimer des composants Plesk, /current/redirect.html?book=administrator-guide&page=74719.htm) (Outils & Paramètres > Mises à jour et à niveau).
  3. Désactivez l'intégration avec un DNS tiers à l'aide de la commande :

    plesk bin server_dns -—disable-custom-backend

Problème de transfert d'abonnement

Si vous transférez des abonnements de Plesk avec un service DNS local vers Plesk avec un service DNS externe, les zones DNS des domaines ne sont pas transférées automatiquement vers les serveurs de noms externes. Vous devez créer manuellement les zones sur les serveurs de noms.
