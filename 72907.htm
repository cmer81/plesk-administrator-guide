Plesk sans serveur DNS

Vous pouvez exclure le composant Serveur DNS de votre installation Plesk. Pour cela, utilisez une installation de type personnalisé (voir le Guide de déploiement) et désélectionnez le composant correspondant (Prise en charge du serveur DNS BIND sous Linux et Serveur DNS Microsoft sous Windows).

Si vous procédez ainsi sans connexion à un service DNS externe, Plesk n'assure pas le service DNS pour les sites Web hébergés dessus. En outre, Plesk réagira comme suit :

  • Le lien vers le Template DNS ne s'affiche pas sur la page Outils & Paramètres du Panneau d'administration du serveur.
  • Dans le Panneau Client, les Paramètres DNS sous Sites Web & Domaines > <nom_de_domaine> sont remplacés par le lien Informations WHOIS. Ce lien dirige les utilisateurs vers la page contenant des informations sur l'enregistrement du nom de domaine.
  • Les utilisateurs de Plesk ne sont pas en mesure d'installer des applications Web qui requièrent des droits de gestion pour les zones DNS (ou aspect DNS).
  • Plesk renvoie des erreurs lors des tentatives de gestion du serveur DNS ou des zones DNS à l'aide de l'utilitaire de ligne de commande dns ou des requêtes XML API avec les nodes <dns>.

Important : s'il y a déjà des domaines sur votre serveur Plesk et si vous installez un serveur DNS ou connectez un service DNS externe par la suite, configurez manuellement les zones DNS de ces domaines. Les zones DNS pour les domaines créés après configuration du service DNS dans Plesk seront créées automatiquement avec le template DNS pour l'ensemble du serveur.
