Envoyer des fichiers de sauvegarde sur un serveur

Pour envoyer un fichier de sauvegarde dans le stockage de Plesk, allez sous Outils & Paramètres > Gestionnaire de sauvegardes et cliquez sur Envoyer. Avant de démarrer l'envoi, Plesk vous invitera à spécifier les paramètres de sauvegarde suivants :

  • Fichier de sauvegarde. Cliquez sur Choisir le fichier pour sélectionner le fichier de sauvegarde que vous voulez envoyer. Par défaut, la taille maximum du fichier est limitée à deux gigaoctets. Si vous avez besoin d’envoyer un fichier plus gros, changez la taille maximum du fichier transféré dans les paramètres Plesk (vous trouverez plus d'informations dans le Guide d'administration avancée (Windows) et le Guide d'administration avancée (Linux)).
  • Envoyer les fichiers de sauvegarde sans signature valide. Si cette case est décochée (option par défaut), Plesk vérifie que le fichier de sauvegarde a une structure et une signature valide et ne le restaure pas si l’un des problèmes suivants est constaté :
    • Le fichier est corrompu.
    • Le fichier a été modifié manuellement après avoir été téléchargé à partir du serveur.
    • Le fichier de sauvegarde a été créé sur un autre serveur.
    • Le fichier a été créé dans une version Plesk antérieure à 11.5.

    Cocher cette case permet de restaurer des données à partir de tels fichiers. Sélectionnez cette option uniquement si la source de la sauvegarde est fiable, car l'envoi d'un tel fichier risque de compromettre la sécurité ou de nuire au fonctionnement du serveur.

    Remarque : les fichiers de sauvegarde créés dans les versions Plesk antérieures à 11.5 sont considérés comme douteux, car ils n'ont pas de signature. Restaurez ce type de fichier si la source est fiable.

  • Utiliser la protection par mot de passe. Cochez cette case si vous avez utilisé un mot de passe pour protéger le fichier de sauvegarde et allez spécifier le mot de passe quand vous envoyez le fichier.
  • Mot de passe. Entrez le mot de passe que vous avez utilisé pour protéger les données de sauvegarde. 

    Remarque : si vous ne sélectionnez pas l’option Utiliser la protection par mot de passe pour une sauvegarde protégée par un mot de passe ou si vous fournissez un mot de passe incorrect, Plesk enverra de toute façon la sauvegarde au serveur. Pendant la restauration de la sauvegarde, vous serez invité à entrer de nouveau le mot de passe.
