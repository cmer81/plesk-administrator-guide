Ports utilisés avec Plesk

Cette section présente les informations sur la configuration du pare-feu intégré à votre panneau de contrôle pour autoriser l'accès à Plesk et à ses services.

Voici une liste des ports et protocoles utilisés par les services de Plesk.

Nom du service

Ports utilisés par le service

Interface administrative de Plesk via HTTPS

TCP 8443

Interface administrative de Plesk via HTTP

TCP 8880

Samba (fichier partageant les réseaux Windows)

UDP 137, UDP 138, TCP 139, TCP 445

Service VPN

UDP 1194

Serveur Web

TCP 80, TCP 443

Serveur FTP

TCP 21

Serveur SSH (shell sécurisé)

TCP 22

Serveur SMTP (envoi de mails) 

TCP 25, TCP 465

Serveur POP3 (récupération des mails)

TCP 110, TCP 995

Serveur IMAP (récupération des mails) 

TCP 143, TCP 993

Service de changement du mot de passe de la messagerie

TCP 106

Serveur MySQL 

TCP 3306

Serveur MS SQL

TCP 1433

Serveur PostgreSQL

TCP 5432

Connexions au serveur de licences

TCP 5224

Serveur de noms de domaine

UDP 53, TCP 53

Programme d'installation de Plesk, mises à jour et à niveau de Plesk

TCP 8447

  

Remarque : si vous installez Presence Builder en tant qu'élément de Plesk, Presence Builder utilise le même protocole et l'ouvre sur le même port que l'interface Plesk.
