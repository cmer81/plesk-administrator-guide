Ajuster les paramètres Apache pour les hôtes virtuels

Vous pouvez personnaliser la configuration Apache pour un site Web particulier dans le Panneau Client sur la page Sites Web & Domaines > <nom_de_domaine> > Paramètres d'Apache et Nginx.

Personnaliser les paramètres communs Apache

La section Paramètres communs Apache contient les paramètres que les propriétaires de sites Web souhaitent généralement ajuster. Par exemple, pour ajouter des fichiers index ou restreindre l'accès au site par adresse IP. Pour chaque paramètre, vous pouvez saisir une valeur personnalisée ou utiliser la configuration Apache par défaut (en sélectionnant la valeur Par défaut).

Remarque : contrairement aux autres paramètres de serveur Web, le paramètre Refuser l'accès au site n'écrase pas la liste des adresses IP fournies par la configuration par défaut, car elle la complète. En cas de conflit (par exemple, si vous autorisez une adresse IP refusée par la configuration par défaut), vos valeurs seront utilisées.

Ajuster des directives supplémentaires Apache

Afin d'ajouter des directives Apache pour un site Web qui ne sont pas disponibles dans les Paramètres communs Apache, utilisez les champs Directives supplémentaires pour HTTP et Directives supplémentaires pour HTTPS. Lorsque vous éditez les champs, utilisez la syntaxe comme dans httpd.conf. Par exemple, si vous souhaitez définir une page d'erreur personnalisée, ajoutez la ligne :

ErrorDocument 401 /my_error_page.html

Important : vos clients ne peuvent ni voir ni éditer ces champs.
