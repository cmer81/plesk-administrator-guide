Utiliser Plesk sans serveur de messagerie

L'utilisation d'un serveur de messagerie dans Plesk est facultative. Avec Plesk pour Linux vous pouvez désactiver ou désinstaller le service de messagerie pour tous les domaines hébergés sur votre serveur. Avec un hébergement Windows, vous ne pouvez pas désactiver le serveur de messagerie par défaut. Toutefois, vous pouvez modifier la configuration du serveur pour interdire les mails sortants. Vous pouvez également exclure le serveur de messagerie des composants installés pendant l'installation de Plesk.

Toutefois, si vous n'installez pas le serveur de messagerie ou si vous interdisez l'envoi de messages sortants, le problème suivant se pose : Plesk doit envoyer des notifications et certains scripts de vos clients peuvent requérir l'envoi de mails. Pour résoudre ce problème, Plesk peut envoyer les mails sortants via un serveur SMTP externe arbitraire.

 

Utilisation d'un serveur SMTP externe pour les mails sortants (Linux)

Lorsque le serveur de messagerie géré par Plesk n'est pas installé, Plesk utilise le client SMTP intégré pour envoyer des mails via le serveur SMTP externe indiqué. Par défaut, le client n'est pas installé et le lien Serveur SMTP externe n'est pas disponible sous Outils & Paramètres > dans le groupe Mail. Plesk vous permet d'installer le client uniquement à la place du serveur de messagerie géré par Plesk.

Pour utiliser le serveur SMTP externe :

  1. Désinstallez le serveur de messagerie Plesk et installez le client SMTP : Dans Outils & Paramètres > Mises à jour et à niveau > Ajouter/Supprimer des composants > Mail hosting, sélectionnez MSMTP (relay only) à la place du serveur de messagerie sélectionné.


    Une fois le serveur de messagerie Plesk désinstallé, les clients ne peuvent plus utiliser les services de messagerie. Pour en savoir plus, consultez la section Fonctionnalités du Panneau Client sans serveur de messagerie ci-dessous.

  2. Définissez le serveur SMTP dans Outils & Paramètres > Serveur SMTP externe (groupe Mail) et sélectionnez au moins l'une de ces options :
    • Autoriser Plesk à envoyer des notifications par mail via ce serveur SMTP
    • Autoriser le script de l'utilisateur à envoyer des mails via ce serveur SMTP

    Autres moyens de configurer les paramètres du serveur SMTP externe :

    • Utilitaire de ligne de commande mailserver (commande --update-smtp-settings)
    • Requêtes API XML avec l'opérateur server (opération set.prefs)
  3. Si le serveur SMTP requiert une authentification, vous devrez communiquer les identifiants d'accès aux propriétaires de sites (Nom d'utilisateur et Mot de passe) afin qu'ils puissent les insérer dans leurs scripts et permettre l'envoi de mails.

Plesk envoie des notifications et mails générés par les scripts via le serveur SMTP externe indiqué. Remarque : si vous n'indiquez pas les paramètres du serveur SMTP externe, aucun service de messagerie n'est disponible.

Une fois que vous avez installé le client, seul le lien - Serveur SMTP externe - sera visible dans Outils & Paramètres, groupe Mail.

Remarque : si vous excluez le serveur de messagerie de la liste de composants lors de l'installation de Plesk, le client SMTP est automatiquement installé à la place du logiciel de serveur de messagerie. Dans ce cas, vous serez invité à fournir les paramètres du client SMTP pendant l'installation de Plesk.

 

Utilisation d'un serveur SMTP externe pour les mails sortants (Windows)

Plesk pour Windows intègre un client SMTP qui envoie les mails sortants au serveur SMTP indiqué dans Outils & Paramètres > Serveur SMTP externe (groupe Mail).

Par défaut, l'option Utiliser le serveur de messagerie installé avec Plesk et le compte de messagerie par défaut créés par Plesk est activée. Autrement dit, le serveur de messagerie installé avec Plesk est utilisé en tant que serveur SMTP externe. Ainsi, Plesk envoie tous les mails sortants via le serveur de messagerie Plesk. Plesk créé automatiquement un compte de messagerie avec des droits administrateur pour accéder au serveur de messagerie installé en local. Le mot de passe de ce compte n'est pas visible sur cette page.

Lorsque vous interdisez les mails sortants depuis le serveur de messagerie Plesk, vous pouvez configurer un autre serveur SMTP afin de permettre à Plesk d'envoyer des notifications et des mails générés par les scripts hébergés :

  1. Indiquez un autre serveur SMTP et un compte d'utilisateur sous Outils & Paramètres > Serveur SMTP externe (groupe Mail). Pour afficher les paramètres du serveur SMTP, décochez l'option Utiliser le serveur de messagerie installé avec Plesk et le compte de messagerie par défaut créés par Plesk.
  2. Sélectionnez l'option Autoriser Plesk à envoyer des notifications par mail via ce serveur SMTP. Plesk enverra des notifications via le serveur SMTP externe indiqué.
  3. Pour autoriser l'envoi de mails par les scripts hébergés, informez les propriétaires des sites que leurs scripts doivent utiliser les paramètres de serveur SMTP à jour.

Si vous n'indiquez pas les paramètres du serveur SMTP externe, aucun service de messagerie sortant n'est disponible.

 

Fonctionnalités du Panneau Client sans serveur de messagerie

Lorsque le serveur de messagerie est désinstallé, Plesk ne fournit aucune fonctionnalité de messagerie à vos abonnés. L'onglet Mail ainsi que les éléments de l'interface liés aux boîtes mail ne sont pas disponibles. En outre, lorsqu'aucun serveur de messagerie n'est installé, les modifications suivantes ont lieu :

  • La messagerie Web n'est pas disponible.
  • Les utilisateurs ne peuvent pas installer les applications APS qui nécessitent un service de messagerie.

Remarque :tous les fichiers et dossiers contenant les données de messagerie de vos abonnés restent sur le serveur Plesk.

Important : lorsque vous transférez des domaines depuis Plesk avec un service de messagerie vers Plesk sans service de messagerie, toutes les données relatives aux boîtes mail des domaines ne sont pas transférées.
