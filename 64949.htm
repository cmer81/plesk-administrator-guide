Exécuter Plesk derrière un routeur avec NAT

Si votre serveur Plesk est derrière un NAT, vous pouvez associer des adresses IP privées sur le serveur aux adresses IP publiques correspondantes via l'interface de Plesk. De cette manière, au lieu d'afficher des adresses IP privées, les adresses IP publiques correspondantes sont affichées dans l'interface de Plesk. Cela permet de rendre l'existence du NAT transparente pour les clients et les revendeurs.

Pour faire correspondre une adresse IP publique avec une adresse IP privée, allez dans Outils & Paramètres > Adresses IP. Cliquez sur l'adresse IP que vous voulez faire correspondre à une adresse IP publique. Saisissez l'adresse IP publique correspondante dans le champ Adresse IP publique.

Après association d'adresses IP publiques à des adresses privées, tous les enregistrements A des zones DNS de tous les domaines sont automatiquement remappés vers les adresses IP publiques correspondantes et les enregistrements A des zones DNS des nouveaux domaines pointent vers les adresses IP (publiques) correspondantes après création du domaine.
